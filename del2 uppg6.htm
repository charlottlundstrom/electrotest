2. Kompilering av electrotest med Makefile

Vi som jobbat tillsammans med denna uppgift är Kai Weeks, Charlott Lundström och Joakim Strandberg.

Huvudprogrammet använder sig av de olika bibliotekens h-fil för att få in funktionsdefinitionerna, sedan länkas biblioteken in vid bygget av huvudprogrammet. Växlarna som använts är de som redovisas nedan. Makefilen återfinns i det bifogade katalogträdets rot. Vi har följande byggmöjligheter:

make lib :

De tre bibliotekskomponenterna kompileras och länkas, var och en med samma kommandon och växlar som beskrivs nedan.

make all:

Huvudprogrammet electrotest kompileras och länkas på samma sätt som beskrivs i del 1 men med alla tre bibliotekskomponenterna (-l) och sökvägar till alla includefiler (-I) på kommandoraden.

make install:

Här kopierar du både programmet och biblioteken till lämpliga kataloger (tex /usr/bin/ och /usr/lib/) och länkar så att programmet använder de publika biblioteken.

make clean:

Tar bort lokala objektsfiler, libfiler och programfiler från arbetskatalogen.

 

 

--Kompilera bibliotek -såhär kanske vi kan göra (----OBS EJ TESTAT, bara på mitt egna projekt!!!)

$ gcc -fPIC -Wall -ansi -g -c calc_component.c calc_power_r.c libresistance.c

$ gcc -fPIC -Wall -ansi -g -c libcomponent.c

Här byggs objektfilerna upp utifrån funktionerna.

 

Nästa steg bygger upp biblioteken, dvs .so-filerna som används i programmet.

$ gcc -shared -fPIC -o libpower.so  calc_power_r.o

$ gcc -shared -fPIC -o libcomponent.so  libcomponent.o

$ gcc -shared -fPIC -o libreststance.so  libresistance.o

 

Kompilera applikation -

$ gcc -o electrotest main.c -L. -lpower -lcomponent lresistance -Wl,-rpath,.

I detta steg skapas den exekverbara filen electrotest.

 

Options/växlar:

-g

Bygg med debuginformation så att det är möjligt att felsöka med gdb om det inte fungerar

-c

Berättar för kompilatorn att montera och kompilera men INTE länka objektfilerna.

-fPIC

Står för Position Independent Code och innebär att den genererade maskinkoden inte är beroende av att lokaliseras på en specifik adress för att fungera. Detta innebär att biblioteken kan flyttas.

-o

Används för att namnge GCCs output, dvs namnet på det körbara programmet.

-Isrc

Lägg till katalogen src för att leta efter include-filer

-L

Titta efter biblioteksfiler i den här katalogen

-l

Länka in det här biblioteket, förutsätter att biblioteket börjar med lib och slutar med .so

-Wl

Används för att skicka parametrar till länkaren, i det här fallet -rpath . , vilket säger till länkaren var den ska börja leta efter .so-filer

-shared

Producerar ett delat objekt som sedan kan kopplas till andra objekt för att bilda en körbar fil.

-Wall

Redovisar eventuella felmeddelanden.

-rpath

Anger rpath som växel till länkaren. Om växeln innehåller komman så delas det upp i flera växlar. rpath står för Run Time Search Path och används för att hitta det aktuella biblioteket.

-ansi

Val av teckenkodning som GCC ska acceptera

 

Ovan nämnda steg i GCC kan också genomföras med make genom all och lib. All skapar den exekverbara filen electrotest. Kommandot lib skapar applikationens olika bibliotek.

 

Hur biblioteken används i huvudprogrammet:

Bibliotek 1 används i huvudprogrammet till att räkna ut den totala resistansen för serie och parallellkopplade resistanser som den sedan returnerar till huvudprogrammet.

Bibliotek 2 används för att räkna ut den totala effekten med hjälp av spänning och ström eller spänning och motstånd när en resistans är kopplad i serie.

Bibliotek 3 innehåller en funktion som används i huvudprogrammet för att beräkna vilka 3 seriekopplade resistorer ur E12 serien som kan ersätta de som användaren angett.
